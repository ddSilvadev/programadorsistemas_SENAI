DER - Diagrama Entidade-Relacionamento

Criamos 3 tabelas para trabalhar com DER

Aluno - guarda dados sobre o estudante
Curso - Armazena os cursos oferecidos pela escola (nome e carga horária).
Matricula - Armazena os cursos oferecidos pela escola (nome e carga horária).


PK — Chave Primária FK — Chave Estrangeira I:N — Um para Muitos N:N resolvido via tabela Matrícula

Matricula usa duas chaves estrangeiras

Curso pode ter muitos alunos matriculados
Matriculas pode ter varios alunos mas apenas uma matricula por aluno


-----------------------
-- Cria o banco de dados com charset UTF-8 de 4 bytes
-- utf8mb4 suporta emojis e caracteres especiais (ex: japonês)
-- COLLATE utf8mb4_unicode_ci = ordenação sem distinção maiúsculas/minúsculas
CREATE DATABASE db_senai
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- Seleciona o banco para que os próximos comandos sejam executados nele
USE db_senai;

-----------------
-- Tabela que armazena os dados de cada aluno
CREATE TABLE aluno (

  -- Chave primária: identifica cada aluno de forma única
  -- AUTO_INCREMENT gera o número automaticamente (1, 2, 3…)
  id_aluno  INT PRIMARY KEY AUTO_INCREMENT,

  -- Nome completo do aluno — campo obrigatório (NOT NULL)
  nome      VARCHAR(100) NOT NULL,

  -- E-mail — opcional (pode ficar vazio/NULL)
  email     VARCHAR(100),

  -- Data de nascimento no formato AAAA-MM-DD
  dt_nasc   DATE

);


-- Tabela que armazena os cursos disponíveis
USE db_senai;
CREATE TABLE curso (

  -- Chave primária do curso, gerada automaticamente
  id_curso       INT PRIMARY KEY AUTO_INCREMENT,

  -- Nome do curso — obrigatório
  nome_curso     VARCHAR(100) NOT NULL,

  -- Carga horária total do curso (em horas)
  carga_horario  INT

);


-- Tabela que liga alunos a cursos (relacionamento N:N)
-- Um aluno pode ter várias matrículas; um curso pode ter vários alunos
USE db_senai;
CREATE TABLE matricula (

  -- Chave primária da matrícula
  id_matricula   INT PRIMARY KEY AUTO_INCREMENT,

  -- Chave estrangeira: referencia qual aluno se matriculou
  id_aluno       INT NOT NULL,

  -- Chave estrangeira: referencia em qual curso
  id_curso       INT NOT NULL,

  -- Data em que a matrícula foi realizada
  data_matricula DATE,

  -- Nota final do aluno no curso (ex: 8.5, 10.0)
  -- DECIMAL(4,1) = até 4 dígitos, sendo 1 decimal (máx 999.9)
  nota           DECIMAL(4,1),

  -- Garante integridade referencial: não aceita id_aluno inexistente
  FOREIGN KEY (id_aluno) REFERENCES aluno(id_aluno),

  -- Garante integridade referencial: não aceita id_curso inexistente
  FOREIGN KEY (id_curso) REFERENCES curso(id_curso)

);



CHARACTER SET utf8mb4 garante suporte a acentos, cedilhas e até emojis. O COLLATE unicode_ci faz buscas sem diferenciar maiúsculas de minúsculas.
Chaves primárias (PRIMARY KEY AUTO_INCREMENT) Presente nas três tabelas. Identifica cada registro de forma única e o valor é gerado automaticamente pelo banco — você não precisa informar na hora do INSERT.
NOT NULL Torna o campo obrigatório. Se tentar inserir um registro sem esse campo, o banco recusa a operação.
DECIMAL(4,1) Armazena números com precisão controlada. DECIMAL(4,1) aceita até 4 dígitos no total, sendo 1 decimal — por exemplo: 10.0, 8.5, 100.0.
Chaves estrangeiras (FOREIGN KEY) São o coração do relacionamento entre tabelas. Elas garantem a integridade referencial: você não consegue matricular um aluno que não existe, nem em um curso inexistente. A tabela matricula depende das outras duas.
A estrutura forma um modelo clássico de banco escolar com três entidades relacionadas.
